## Main Issues Found

- Some wording sounded too commercial for the Apache APISIX open-source website.
- Several pages used "best" or "commercial" style language that could imply absolute rankings or sales positioning.
- AI Gateway claims needed explicit APISIX source notes.
- The page template had CTAs, related guides, and source notes, but lacked a clear open-source next-step section for docs, plugins, GitHub, community, and contribution paths.
- Several pages still used migration wording after the migration cluster was removed.

## Changes Completed

- Removed commercial/sales-oriented language and replaced it with open-source evaluation wording.
- Removed API7/API7 Enterprise/API7 AI Gateway references from the current page set.
- Replaced migration language with rollout/evaluation language.
- Added an "Open-source next steps" section to all comparison and alternative pages generated by the shared template.
- Added links to Apache APISIX docs, plugin hub, Ingress Controller docs, GitHub, community, and contribution pages.
- Added APISIX AI Gateway and `ai-proxy` plugin source notes where AI Gateway capabilities are discussed.
- Kept the module to two user-facing hub clusters: "Compare API Gateways" and "Explore Alternatives".
- Preserved the distinction between Apache APISIX Gateway and Apache APISIX Ingress Controller.
- Preserved the distinction between Kong Gateway and Kong Ingress Controller.

## SEO Report Issues Addressed

- Strengthened comparison and alternative pages for non-branded, evaluation-oriented search intent.
- Improved SERP intent fit through clearer page positioning, metadata, and H1 alignment.
- Reduced duplicate/cannibalization risk by keeping category pages, direct comparison pages, and alternative pages distinct.
- Improved internal linking to open-source resources instead of commercial conversion paths.
- Added clearer next steps for developers, platform engineers, Kubernetes users, AI Gateway evaluators, and community contributors.

## Not Changed

- Homepage, navbar, footer, global templates, sitemap, robots, redirects, docs, blog, and learning center pages were not modified in this review.
- Existing page slugs were not changed.
- No new comparison/alternative pages were added.
